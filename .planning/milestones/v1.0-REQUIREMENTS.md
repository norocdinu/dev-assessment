# Requirements Archive: v1.0

**Archived:** 2026-05-07
**Status:** All 24 v1 requirements shipped and verified.

---

## v1 Requirements — All Complete

### Assessment Delivery (ASSESS)

- [x] **ASSESS-01**: Candidate can open a pre-configured link and start a timed MCQ test immediately (no login, no signup) — *Phase 2*
- [x] **ASSESS-02**: Test runs for exactly 30 minutes; timer is enforced server-side and auto-submits on expiry — *Phase 2*
- [x] **ASSESS-03**: Candidate can see a live countdown timer with visual urgency cues (green → yellow → red) — *Phase 2*
- [x] **ASSESS-04**: Candidate can navigate between questions, change answers, and submit early before timer expires — *Phase 2*
- [x] **ASSESS-05**: If candidate refreshes or reopens the link mid-test, the test resumes with remaining time intact — *Phase 2*
- [x] **ASSESS-06**: Questions are randomly drawn from the pool for the configured technology and seniority level, using a seeded algorithm so the same link always shows the same questions — *Phase 2*

### Question Bank (QBANK)

- [x] **QBANK-01**: Admin can create, edit, and delete MCQ questions with: question text, 4 answer options, correct answer, skill-area tag, difficulty level (Junior/Mid/Senior), and technology tag — *Phase 1*
- [x] **QBANK-02**: Admin can search and filter questions by technology, difficulty, and skill area — *Phase 1*
- [x] **QBANK-03**: Admin can bulk-import questions from a CSV file with error reporting per row — *Phase 4*
- [x] **QBANK-04**: Edited questions create a new version; past submissions remain linked to the version that was active at submission time — *Phase 1*
- [x] **QBANK-05**: Admin can soft-delete (archive) questions; archived questions are excluded from new tests but visible for audit — *Phase 1*

### Test Configuration (TESTS)

- [x] **TESTS-01**: Admin can create a test configuration specifying: technology, seniority level, number of questions to draw, pass/fail threshold (%) — *Phase 1*
- [x] **TESTS-02**: Admin can generate a shareable link for a test configuration; the link encodes a seed so question selection is deterministic — *Phase 2*
- [x] **TESTS-03**: Admin can set an expiry date on a link; expired links show a clear message to candidates — *Phase 2*
- [x] **TESTS-04**: Platform supports any technology (Power BI, SFMC, Data Engineering, etc.) — technology is a configurable tag, not hardcoded — *Phase 2*
- [x] **TESTS-05**: Platform supports Junior, Mid, and Senior seniority levels per technology; new levels can be added without code changes — *Phase 2*

### Auto-Grading & Results (GRADE)

- [x] **GRADE-01**: System auto-grades MCQ submissions immediately on receipt; no manual grading required for v1 — *Phase 3*
- [x] **GRADE-02**: Candidate can view their results after submitting: total score (%), pass/fail verdict, and time taken — *Phase 3*
- [x] **GRADE-03**: Candidate results show a skill-area breakdown: score per tagged topic (e.g., "DAX: 70%, Data Modelling: 90%") — *Phase 3*
- [x] **GRADE-04**: Candidate results show the full answer sheet: each question, the candidate's answer, the correct answer, and whether it was correct — *Phase 3*

### Admin Dashboard (DASH)

- [x] **DASH-01**: Admin can view a list of all submissions for a test, sortable and filterable by score, date, and seniority level — *Phase 4*
- [x] **DASH-02**: Admin can drill into any submission to see the full result detail (score, breakdown, answer sheet) — *Phase 3 + 4*
- [x] **DASH-03**: Admin can compare multiple candidates side-by-side: scores, skill breakdowns, time taken — *Phase 4*
- [x] **DASH-04**: Dashboard shows aggregate statistics per test: average score, pass rate, score distribution — *Phase 4*
- [x] **DASH-05**: Admin can export submission results to CSV — *Phase 4*

### Access & Admin (ACCESS)

- [x] **ACCESS-01**: Admin users log in with email + password; candidates never need an account — *Phase 1*
- [x] **ACCESS-02**: Multiple admins can manage the question bank and view results — *Phase 1*
- [x] **ACCESS-03**: Admin roles: Owner (full access) and Reviewer (view results only, no question editing) — *Phase 1*
- [x] **ACCESS-04**: All question create/edit/delete actions are logged with admin identity and timestamp — *Phase 1*

## Out of Scope — Validated

- Open-ended / coding questions — MCQ auto-grading sufficient for v1 screening ✓
- Candidate login/accounts — link-based zero-friction access validated ✓
- Video proctoring — live interview setting makes this unnecessary ✓
- Mobile native app — responsive web covers the use case ✓
- Real-time collaboration on question editing — CRDT complexity for negligible gain ✓
