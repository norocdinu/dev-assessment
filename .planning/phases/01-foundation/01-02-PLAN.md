## Objective

Build the complete backend API: admin authentication (email/password → JWT), role-based access control (Owner vs Reviewer), question bank CRUD with versioning, soft-delete, audit logging, question search/filter, and test configuration management.

## Context

- Framework: Fastify 4 + TypeScript
- Auth: httpOnly cookie containing a signed JWT (`@fastify/jwt`, `@fastify/cookie`)
- RBAC: middleware checks `req.user.role` before mutating routes
- DB client: singleton `postgres` from `apps/api/src/db/client.ts`
- Versioning: editing a question creates a new row with `version+1`, sets `is_latest=FALSE` on old row
- Audit: every mutating question action inserts a row into `audit_log`

### Task 1 — Admin auth routes

- `apps/api/src/db/client.ts` (db singleton import)

<action>
Create `apps/api/src/middleware/auth.ts`:
- Fastify preHandler that reads the JWT from the `token` httpOnly cookie
- Verifies with `fastify.jwt.verify()`; on failure returns 401 `{ error: 'Unauthorized' }`
- Attaches decoded payload to `req.user = { id, email, role }`

Create `apps/api/src/middleware/rbac.ts`:
- Export `requireRole(...roles: string[])` — returns a Fastify preHandler
- If `req.user.role` not in `roles`, returns 403 `{ error: 'Forbidden' }`

Create `apps/api/src/routes/auth.ts` with these endpoints:

**POST /auth/login**
- Body: `{ email: string, password: string }` (validate with Zod)
- Query `admin_users` by email; if not found → 401 `{ error: 'Invalid credentials' }`
- `bcrypt.compare(password, row.password_hash)`; if false → 401 same error
- Update `last_login_at = NOW()`
- Sign JWT: `{ id, email, role }`, expires in `JWT_EXPIRES_IN` env var (default `8h`)
- Set httpOnly cookie `token` with `sameSite: 'lax'`, `path: '/'`
- Return `{ user: { id, email, role } }`

**POST /auth/logout**
- Protected (requires auth middleware)
- Clear the `token` cookie
- Return `{ ok: true }`

**GET /auth/me**
- Protected
- Return `{ user: { id, email, role } }`

Seed script `apps/api/src/db/seed-admin.ts`:
- Creates a default owner: `email=admin@example.com`, `password=Admin1234!`
- Uses `bcrypt.hash(password, 12)`
- Skips if email already exists
- Run with: `npx ts-node src/db/seed-admin.ts`
</action>

<acceptance_criteria>
- `POST /auth/login` with correct credentials returns 200 and sets a `token` cookie
- `POST /auth/login` with wrong password returns 401
- `GET /auth/me` without cookie returns 401
- `GET /auth/me` with valid cookie returns `{ user: { id, email, role } }`
- `apps/api/src/middleware/rbac.ts` exports `requireRole`
- `requireRole('owner')` returns 403 when called with a reviewer token
</acceptance_criteria>

### Task 2 — Question bank API (CRUD + versioning + soft-delete)

<read_first>
- `apps/api/src/db/schema.sql` (questions table — family_id, version, is_latest, is_active columns)
- `apps/api/src/middleware/auth.ts`
- `apps/api/src/middleware/rbac.ts`
- `apps/api/src/lib/audit.ts` (write this in Task 3 first, then import here)
</read_first>

<action>
Create `apps/api/src/routes/questions.ts` — all routes require auth middleware; mutating routes also require `requireRole('owner')`.

**GET /questions**
Query params (all optional): `technology` (slug), `difficulty` ('junior'|'mid'|'senior'), `skill_area` (partial match), `search` (full-text on `text`), `include_archived` (boolean, default false)

SQL: `SELECT q.*, t.name AS technology_name FROM questions q JOIN technologies t ON t.id = q.technology_id WHERE q.is_latest = TRUE [AND q.is_active = TRUE if not include_archived] [AND t.slug = $technology] [AND q.difficulty = $difficulty] [AND q.skill_area ILIKE $skill_area] [AND q.text ILIKE $search] ORDER BY q.created_at DESC`

Returns: array of question objects including `technology_name`, `family_id`, `version`.

**GET /questions/:familyId/versions**
Returns all versions of a question family ordered by version DESC. Used for version history view.

**POST /questions**
- Requires `requireRole('owner')`
- Body (Zod): `{ technology_id, difficulty, skill_area, text, option_a, option_b, option_c, option_d, correct_option, explanation? }`
- Generate `family_id = uuid()`
- Insert with `version=1, is_latest=TRUE, is_active=TRUE, created_by=req.user.id`
- Insert audit_log: `action='question.create'`, `entity_type='question'`, `entity_id=new_question.id`
- Return 201 with created question

**PUT /questions/:familyId**
- Requires `requireRole('owner')`
- Body: same as POST (all fields optional — only provided fields change)
- Fetch current latest version for `family_id` where `is_latest=TRUE`
- If not found → 404
- In a transaction:
  1. `UPDATE questions SET is_latest=FALSE WHERE family_id=$1 AND is_latest=TRUE`
  2. `INSERT questions (..., version=current.version+1, is_latest=TRUE, family_id=current.family_id, created_by=req.user.id)` with updated fields merged over current values
- Insert audit_log: `action='question.edit'`, `entity_id=new_version.id`, `detail={ from_version: N, to_version: N+1 }`
- Return 200 with new version

**DELETE /questions/:familyId**  (soft-delete / archive)
- Requires `requireRole('owner')`
- `UPDATE questions SET is_active=FALSE WHERE family_id=$1 AND is_latest=TRUE`
- Insert audit_log: `action='question.archive'`, `entity_id=familyId`
- Return `{ ok: true }`
</action>

<acceptance_criteria>
- `POST /questions` with owner token returns 201 and creates a row with `version=1, is_latest=TRUE`
- `PUT /questions/:familyId` creates a new row with `version=2, is_latest=TRUE` and sets old row `is_latest=FALSE`
- `DELETE /questions/:familyId` sets `is_active=FALSE` on the row; row still exists in DB
- `GET /questions` with `include_archived=false` (default) does not return archived questions
- Every mutating call inserts a row in `audit_log` with correct `action` value
- `POST /questions` with reviewer token returns 403
</acceptance_criteria>

### Task 3 — Audit log helper

<read_first>
- `apps/api/src/db/schema.sql` (audit_log table columns)
- `apps/api/src/db/client.ts`
</read_first>

<action>
Create `apps/api/src/lib/audit.ts`:

```typescript
import { db } from '../db/client';

export async function logAudit(params: {
  adminId: string;
  action: string;         // e.g. 'question.create'
  entityType: string;     // e.g. 'question'
  entityId: string;
  detail?: Record<string, unknown>;
}): Promise<void> {
  await db`
    INSERT INTO audit_log (admin_id, action, entity_type, entity_id, detail)
    VALUES (${params.adminId}, ${params.action}, ${params.entityType}, ${params.entityId}, ${params.detail ?? null})
  `;
}
```
</action>

<acceptance_criteria>
- `apps/api/src/lib/audit.ts` exports `logAudit`
- After any question create/edit/archive, `SELECT COUNT(*) FROM audit_log` increases by 1
- `audit_log` row contains correct `admin_id`, `action`, `entity_type`, `entity_id`
</acceptance_criteria>

### Task 5 — Wire all routes into Fastify app

<read_first>

<action>
Update `apps/api/src/index.ts`:

```typescript
import Fastify from 'fastify';
import fastifyJwt from '@fastify/jwt';
import fastifyCookie from '@fastify/cookie';
import fastifyCors from '@fastify/cors';
import { authRoutes } from './routes/auth';
import { questionRoutes } from './routes/questions';
import { testConfigRoutes } from './routes/test-configs';
import { technologyRoutes } from './routes/technologies';

const app = Fastify({ logger: true });

await app.register(fastifyCors, { origin: process.env.WEB_URL ?? 'http://localhost:3000', credentials: true });
await app.register(fastifyCookie);
await app.register(fastifyJwt, { secret: process.env.JWT_SECRET!, cookie: { cookieName: 'token', signed: false } });

await app.register(authRoutes, { prefix: '/auth' });
await app.register(questionRoutes, { prefix: '/questions' });
await app.register(testConfigRoutes, { prefix: '/test-configs' });
await app.register(technologyRoutes, { prefix: '/technologies' });

app.listen({ port: Number(process.env.PORT ?? 3001), host: '0.0.0.0' });
```
</action>
