## Objective

Create the `apps/api/src/routes/dashboard.ts` route file with two endpoints (`GET /dashboard/stats` and `GET /dashboard/competency`) that aggregate cross-config analytics data from `submission_results`. Extend `apps/api/src/routes/submissions.ts` with a `DELETE /:linkId` handler (owner-only, transactional, audit-logged). Register the new dashboard routes in `apps/api/src/index.ts`.

## Must-Haves

These conditions must be true for the phase goal to be met:
- `GET /dashboard/stats` returns `{ totalCandidates, passRate, avgScore, weakestSkillArea, recentSubmissions }` with HTTP 200 (including when there are zero submissions — returns zeros, not 404)
- `GET /dashboard/stats` response includes `recentSubmissions` array of up to 10 items, each with `candidateName`, `scorePct`, `pass`, `submittedAt`, `testConfigName` fields
- `GET /dashboard/stats` response includes 6 distribution buckets: `bucket0_49`, `bucket50_59`, `bucket60_69`, `bucket70_79`, `bucket80_89`, `bucket90_100`
- `GET /dashboard/competency` returns an array of `{ area, avgScore }` objects derived from `skill_area_scores` JSONB using a lateral join
- `DELETE /admin/submissions/:linkId` is guarded by `requireRole('owner')`, returns 404 if no `submission_results` row exists for the linkId, writes an `audit_log` entry before the transaction, deletes `candidate_answers` then `submission_results` in a single `db.begin()` transaction, preserves the `test_links` row, and returns 204
- All three new/modified route files pass `npx tsc --noEmit` in `apps/api/`

```ts
const [stats] = await db`
  SELECT
    COUNT(*)                                                                AS total_candidates,
    COALESCE(ROUND(AVG(sr.score_pct)), 0)                                  AS avg_score_pct,
    COALESCE(ROUND(COUNT(*) FILTER(WHERE sr.pass) * 100.0
          / NULLIF(COUNT(*), 0)), 0)                                        AS pass_rate_pct,
    COUNT(*) FILTER(WHERE sr.score_pct BETWEEN 0  AND 49)                  AS bucket_0_49,
    COUNT(*) FILTER(WHERE sr.score_pct BETWEEN 50 AND 59)                  AS bucket_50_59,
    COUNT(*) FILTER(WHERE sr.score_pct BETWEEN 60 AND 69)                  AS bucket_60_69,
    COUNT(*) FILTER(WHERE sr.score_pct BETWEEN 70 AND 79)                  AS bucket_70_79,
    COUNT(*) FILTER(WHERE sr.score_pct BETWEEN 80 AND 89)                  AS bucket_80_89,
    COUNT(*) FILTER(WHERE sr.score_pct BETWEEN 90 AND 100)                 AS bucket_90_100
  FROM submission_results sr
`;
```

**Query 2 — weakest skill area (separate db call):**
```ts
const [weakest] = await db`
  SELECT
    kv.key                                                       AS area,
    ROUND(AVG((kv.value::jsonb->>'pct')::numeric))               AS avg_score
  FROM submission_results sr,
       jsonb_each(sr.skill_area_scores) AS kv(key, value)

  ORDER BY avg_score ASC

`;
```

**Query 3 — recent submissions (separate db call):**
```ts
const recentSubmissions = await db`
  SELECT
    tl.candidate_name,
    sr.score_pct,
    sr.pass,

    tc.name AS test_config_name
  FROM submission_results sr
  JOIN test_links   tl ON tl.id = sr.link_id
  JOIN test_configs tc ON tc.id = tl.test_config_id
  ORDER BY tl.submitted_at DESC
  LIMIT 10
`;
```

The handler reads an optional `testConfigId` query parameter. When provided, it filters by joining `test_links` and filtering on `tl.test_config_id`. When absent, the query runs globally.

```ts
app.get('/competency', { preHandler: [authMiddleware] }, async (request, reply) => {
  const queryRaw = request.query as Record<string, string>;
  const testConfigId = queryRaw.testConfigId ?? null;

  let rows;
  if (testConfigId) {
    rows = await db`
      SELECT
        kv.key                                                       AS area,
        ROUND(AVG((kv.value::jsonb->>'pct')::numeric))               AS avg_score
      FROM submission_results sr
      JOIN test_links tl ON tl.id = sr.link_id,
           jsonb_each(sr.skill_area_scores) AS kv(key, value)
      WHERE tl.test_config_id = ${testConfigId}
      GROUP BY kv.key
      ORDER BY avg_score DESC
    `;
  } else {
    rows = await db`
      SELECT
        kv.key                                                       AS area,
        ROUND(AVG((kv.value::jsonb->>'pct')::numeric))               AS avg_score
      FROM submission_results sr,
           jsonb_each(sr.skill_area_scores) AS kv(key, value)
      GROUP BY kv.key
      ORDER BY avg_score DESC
    `;
  }

  return reply.status(200).send(
    rows.map(r => ({
      area: r.area,
      avgScore: Number(r.avg_score),
    }))
  );
});
```

```ts
// DELETE /admin/submissions/:linkId  — hard delete (owner-only)
app.delete('/:linkId', { preHandler: [authMiddleware, requireRole('owner')] }, async (request, reply) => {
  const { linkId } = request.params as { linkId: string };

  // 1. Check submission exists
  const [existing] = await db`
    SELECT id FROM submission_results WHERE link_id = ${linkId}
  `;
  if (!existing) {
    return reply.status(404).send({ error: 'Submission not found' });
  }

  // 2. Write audit log before transaction
  const admin = getAuthUser(request);
  await db`
    INSERT INTO audit_log (admin_id, action, entity_type, entity_id, detail)
    VALUES (
      ${admin.id},
      'submission.delete',
      'submission',
      ${linkId},
      ${JSON.stringify({ deleted_by: admin.email })}::jsonb
    )
  `;

  // 3. Delete in transaction: candidate_answers first (FK), then submission_results
  await db.begin(async sql => {
    await sql`DELETE FROM candidate_answers WHERE link_id = ${linkId}`;
    await sql`DELETE FROM submission_results WHERE link_id = ${linkId}`;
  });

  // test_links row is preserved (D-09)
  return reply.status(204).send();
});
```

The `audit_log.entity_id` column is `UUID NOT NULL` — `linkId` is a UUID (the test_links primary key), which satisfies the constraint.
</action>
