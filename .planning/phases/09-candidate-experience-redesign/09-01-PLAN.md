# Plan 09-01: Design System Foundation

## Objective

Set up the CSS variable token system, enable `darkMode: 'class'` in Tailwind, wire brand colour from env var, and mount `CandidateThemeProvider` in the candidate layout so that dark mode auto-tracks OS preference. This is a prerequisite for all visual changes in Plan 09-02.

<read_first>

</read_first>

<action>
Replace the entire contents of `apps/web/tailwind.config.ts` with:

```ts
import type { Config } from 'tailwindcss';

const config: Config = {

  theme: {
    extend: {
```

</read_first>

<action>

```tsx
import type { Metadata } from 'next';
import './globals.css';

export const metadata: Metadata = {

export default function RootLayout({ children }: { children: React.ReactNode }) {
```

### Task 4: Create CandidateThemeProvider component

<read_first>
- apps/web/src/app/(candidate)/layout.tsx
</read_first>

<action>
Create `apps/web/src/components/candidate/CandidateThemeProvider.tsx`:

```tsx
'use client';

import { useEffect } from 'react';

export function CandidateThemeProvider({ children }: { children: React.ReactNode }) {
  useEffect(() => {
    const mq = window.matchMedia('(prefers-color-scheme: dark)');

    const apply = (e: MediaQueryList | MediaQueryListEvent) => {
      document.documentElement.classList.toggle('dark', e.matches);
    };

    apply(mq);
    mq.addEventListener('change', apply);
    return () => mq.removeEventListener('change', apply);
  }, []);

  return <>{children}</>;
}
```

Then update `apps/web/src/app/(candidate)/layout.tsx` to mount it:

```tsx
import { CandidateThemeProvider } from '@/components/candidate/CandidateThemeProvider';

export default function CandidateLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return <CandidateThemeProvider>{children}</CandidateThemeProvider>;
}
```
</action>

<acceptance_criteria>
- File `apps/web/src/components/candidate/CandidateThemeProvider.tsx` exists
- `CandidateThemeProvider.tsx` contains `window.matchMedia('(prefers-color-scheme: dark)')`

- `CandidateThemeProvider.tsx` contains `mq.addEventListener('change', apply)` and `mq.removeEventListener('change', apply)`
