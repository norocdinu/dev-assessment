# Plan: DB Schema + Shared Types + Migration

## Objective
Add the `submission_results` table to schema.sql, add a Phase 3 migration block to migrate.ts so existing databases upgrade cleanly, and add the `SkillAreaScore`, `AnswerSheetRow`, `AdminAnswerSheetRow`, `SubmissionResult`, and `AdminSubmissionResult` TypeScript types to the shared package. These are pure foundation artefacts — no API or UI code touches them until Wave 2/3.

## must_haves
- `submission_results` table exists in schema.sql with `UNIQUE (link_id)` constraint, `score_pct INT CHECK (score_pct BETWEEN 0 AND 100)`, `pass BOOLEAN`, `skill_area_scores JSONB`, `time_taken_seconds INT`
- migrate.ts checks for `submission_results` table existence and runs Phase 3 DDL if absent, following the identical section-marker pattern used for Phase 2
- `packages/shared/src/types/index.ts` exports `SkillAreaScore`, `AnswerSheetRow`, `AdminAnswerSheetRow`, `SubmissionResult`, `AdminSubmissionResult` with all fields documented in RESEARCH.md section 7
